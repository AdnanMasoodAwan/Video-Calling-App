\doxysubsection{Detailed Description}
The \mbox{\hyperlink{classagora_1_1rtc_1_1_i_rtc_engine_event_handler}{I\+Rtc\+Engine\+Event\+Handler}} class.

The S\+DK uses this class to send callback event notifications to the app, and the app inherits the methods in this class to retrieve these event notifications.

All methods in this class have their default (empty) implementations, and the app can inherit only some of the required events instead of all. In the callback methods, the app should avoid time-\/consuming tasks or calling blocking A\+P\+Is, otherwise the S\+DK may not work properly.

Reports the volume information of users.

By default, this callback is disabled. You can enable it by calling {\ttfamily enable\+Audio\+Volume\+Indication}. Once this callback is enabled and users send streams in the channel, the S\+DK triggers the {\ttfamily on\+Audio\+Volume\+Indication} callback at the time interval set in {\ttfamily enable\+Audio\+Volume\+Indication}. The S\+DK triggers two independent {\ttfamily on\+Audio\+Volume\+Indication} callbacks simultaneously, which separately report the volume information of the local user who sends a stream and the remote users (up to three) whose instantaneous volume is the highest.

\begin{DoxyNote}{Note}
After you enable this callback, calling mute\+Local\+Audio\+Stream affects the S\+DK\textquotesingle{}s behavior as follows\+:
\begin{DoxyItemize}
\item If the local user stops publishing the audio stream, the S\+DK stops triggering the local user\textquotesingle{}s callback.
\item 20 seconds after a remote user whose volume is one of the three highest stops publishing the audio stream, the callback excludes this user\textquotesingle{}s information; 20 seconds after all remote users stop publishing audio streams, the S\+DK stops triggering the callback for remote users.
\end{DoxyItemize}

When the state of the local video stream changes (including the state of the video capture and encoding), the S\+DK triggers this callback to report the current state. This callback indicates the state of the local video stream, including camera capturing and video encoding, and allows you to troubleshoot issues when exceptions occur.

The S\+DK triggers the on\+Local\+Video\+State\+Changed callback with the state code of {\ttfamily L\+O\+C\+A\+L\+\_\+\+V\+I\+D\+E\+O\+\_\+\+S\+T\+R\+E\+A\+M\+\_\+\+S\+T\+A\+T\+E\+\_\+\+F\+A\+I\+L\+ED} and error code of {\ttfamily L\+O\+C\+A\+L\+\_\+\+V\+I\+D\+E\+O\+\_\+\+S\+T\+R\+E\+A\+M\+\_\+\+R\+E\+A\+S\+O\+N\+\_\+\+C\+A\+P\+T\+U\+R\+E\+\_\+\+F\+A\+I\+L\+U\+RE} in the following situations\+:
\begin{DoxyItemize}
\item The app switches to the background, and the system gets the camera resource.
\item The camera starts normally, but does not output video for four consecutive seconds.
\end{DoxyItemize}

When the camera outputs the captured video frames, if the video frames are the same for 15 consecutive frames, the S\+DK triggers the {\ttfamily on\+Local\+Video\+State\+Changed} callback with the state code of {\ttfamily L\+O\+C\+A\+L\+\_\+\+V\+I\+D\+E\+O\+\_\+\+S\+T\+R\+E\+A\+M\+\_\+\+S\+T\+A\+T\+E\+\_\+\+C\+A\+P\+T\+U\+R\+I\+NG} and error code of {\ttfamily L\+O\+C\+A\+L\+\_\+\+V\+I\+D\+E\+O\+\_\+\+S\+T\+R\+E\+A\+M\+\_\+\+R\+E\+A\+S\+O\+N\+\_\+\+C\+A\+P\+T\+U\+R\+E\+\_\+\+F\+A\+I\+L\+U\+RE}. Note that the video frame duplication detection is only available for video frames with a resolution greater than 200 × 200, a frame rate greater than or equal to 10 fps, and a bitrate less than 20 Kbps.

\begin{DoxyNote}{Note}
